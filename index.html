Secure File Transfer With Prior Key Agreement

One of my assignments from CPSC 418 in the W2020 semester. This assignment assumes that the Client and Server have negotiated SRP-SHA3-256-RSA-AES-256-CBC-SHA3-256 as the TLS cipher suite. At a high-level, this project implements a toy version of the TLS handshake using the cipher suite specified above. The protocol has two parties (a Client and a Server), as well as a Trusted Third Party. The program allows for the Client and Server to register and authenticate each other's identities using certificates, and then allows the users to derive a shared key so that the Client can send encrypted files to the Server.
